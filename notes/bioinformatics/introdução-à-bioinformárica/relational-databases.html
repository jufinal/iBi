Keys to a Relational Nature

  • Primary Key (PK)
    • The exclusive identifier for a single Entry in a specific Table
    • Unique as identifier
  • Foreign Key (FK)
    • The otherwise PK may be used as needed in other Tables as reference to
      • The referenced Entry
      • In the source Table
  • (Table) Join
    • Relation between two Tables on a PK:FK pair

  • Fields / Attributes must be atomic
    • No array storage allowed.
  • Data redundancy is to be avoided at all costs
    • To streamline the data structure
    • To avoid typing errors
  • Standard Language across different RDBMS
    • SQL

      • Selection

      • Projection

      • Join (Tables)

"The world is your oyster"

  • Databases can be accessed seamlessly through the World Wide Web

  • Clusters of different DBMS can be made available through middleware plus client software as single system
